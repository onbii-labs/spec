# Contributing to Onbii

First of all, thank you for your interest in contributing to Onbii.

Onbii is an open specification for portable, local-first personal knowledge. Whether you are fixing a typo, proposing a new concept, improving documentation, or implementing part of the specification, your contributions are welcome.

## Before You Start

Please take a moment to read:

- [`README.md`](./README.MD)
- [`docs/VISION.md`]()
- [`docs/PRINCIPLES.md`]()

Understanding the goals of the project will make it much easier to contribute effectively.

## Ways to Contribute

Contributions are welcome in many forms, including:

- Improving documentation
- Correcting mistakes or ambiguities
- Proposing additions to the specification
- Providing examples
- Discussing design decisions
- Implementing reference tooling
- Reporting issues

## Discuss First

For significant changes, please open a GitHub Discussion or Issue before starting implementation.

Early discussion helps ensure that proposals align with the goals and direction of the project and avoids unnecessary work.

## Pull Requests

When submitting a pull request:

- Keep changes focused on a single topic.
- Explain the motivation behind the change.
- Update documentation where appropriate.
- Be respectful during discussion and review.
- Be open to feedback and iteration.

## Guiding Principles

When making design decisions, we favour solutions that are:

- Local-first
- Open and interoperable
- Human-readable where practical
- Simple over complex
- Stable and backwards compatible whenever possible
- Vendor-neutral

Not every proposal will be accepted, but every proposal will be considered on its technical merits.

## Code of Conduct

By participating in this project, you agree to abide by the project's [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT).

## Questions

If you're unsure about anything, please open a GitHub Discussion.

We'd much rather discuss an idea early than have you spend time building something that doesn't fit the direction of the project.
