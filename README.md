# Onbii Specification

Onbii is an open specification for local-first personal knowledge.

It defines a way to represent knowledge as portable objects that keep source material, human-readable representations, derived artefacts, provenance, and links together. The aim is to let people keep their knowledge understandable and under their control while still allowing different recorders, applications, AI providers, sync tools, and storage systems to interoperate.

This repository contains the specification, design documents, and recovery notes for the early Onbii project.

## Current Status

Onbii is at the design and specification stage.

There is not yet a stable bundle format, schema, CLI, SDK, desktop app, mobile app, or reference implementation. The current work is to define the object model and the principles before creating implementation repositories.

## Design Recovery

The `docs/recovery/` directory contains the design-history recovery work used to reconstruct the current direction from the original project conversation.

These files are not the specification itself. They are source material for the first canonical docs and should be distilled over time.
