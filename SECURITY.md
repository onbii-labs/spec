# Security Policy

## Reporting a Security Vulnerability

The **Onbii Specification** repository primarily contains documentation and specification documents. As such, it is unlikely that security vulnerabilities exist within this repository itself.

If you believe you have identified a security issue relating to:

- a future reference implementation;
- tooling maintained by the Onbii project; or
- the specification itself that could lead to security or privacy risks,

please report it privately rather than opening a public GitHub issue.

You can contact the maintainers by opening a GitHub Private Vulnerability Report (if enabled) or by using the project's published security contact details.

Please include:

- a description of the issue;
- the affected component or specification section;
- steps to reproduce (where applicable);
- the potential impact; and
- any suggested mitigation.

## Responsible Disclosure

Please do not publicly disclose security vulnerabilities until the maintainers have had a reasonable opportunity to investigate and, where appropriate, address the issue.

We appreciate responsible disclosure and will acknowledge valid reports.

## Supported Versions

As the specification evolves, the latest published version should be considered the supported version unless stated otherwise.
