# 0020: Mobile Matters, But Not Necessarily Day One

Status: Accepted

Date: 2026-07-21

## Context

Many people live primarily on phones. Mobile capture, browsing, search, and sync are strategically important if Onbii is to work beyond desktop-oriented users.

At the same time, the early project needs controlled learning around the object model, source material, transcription, bundles, provenance, and basic views.

## Decision

Mobile should be treated as strategically important, but not necessarily part of the first implementation.

Early validation may happen in a more controlled desktop or local workflow, as long as the architecture does not make mobile a later afterthought.

## Consequences

- Roadmap should mention mobile relatively early.
- The object model should not assume a desktop-only workflow.
- Mobile-to-desktop sync remains an important future concern.
- The first producer workflow remains open.
