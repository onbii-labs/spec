# 0027: Future Code License Remains Open

Status: Open

Date: 2026-07-21

## Context

The specification and documentation use CC BY 4.0. Future implementation repositories may contain software such as CLI tools, SDKs, adapters, desktop helpers, mobile clients, or plugins.

The conversation identified Apache 2.0 and MPL 2.0 as plausible software licenses, with different trade-offs around adoption, patent grants, and file-level copyleft.

## Decision

No default license for future code repositories has been chosen yet.

## Consequences

- The `spec` repository license does not decide future implementation licenses.
- New code repositories should choose an appropriate software license before accepting contributions.
- Apache 2.0 and MPL 2.0 remain leading candidates.
- Trademark and brand rights remain separate from both documentation and code licenses.
